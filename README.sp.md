### Estructura

The challenge template has four main parts:

1. [DISClib](./DISClib) Carpeta principal con la librería oficial para la clase. Para obtener más información sobre su implementación, visite el [DISClib Repository][disclib-url].
1. [App](./App) Carpeta con los scripts de Python Modelo-Vista-Controlador (MVC). Aquí los estudiantes implementan su código para completar el reto.
1. [Data](./Data) Carpeta con archivos de datos CSV para cargar en la aplicación. Aquí los estudiantes agregan los archivos de datos proporcionados por el personal del curso para completar el reto.
1. [Docs](./Docs) Carpeta con los informes, tabla de datos y otra documentación. Aquí los estudiantes agregan su informe de proyecto, tablas de datos y otra documentación para complementar su implementación de código.

## Uso
Para usar esta plantilla, debe seguir los pasos a continuación:

* Lea el enunciado oficial del proyecto publicado en el sitio oficial del curso en [BrightSpace][BrightSpace-url].
* Distribuya las funcionalidades las responsabilidades para implementar el proyecto entre los miembros del grupo.
* Descargue los datos oficiales del proyecto en el sitio oficial del curso en[BrightSpace][BrightSpace-url].
* Descomprima y cargue los datos de la aplicación en [Data](./Data) folder.
*  Importe los módulos necesarios desde [DISClib](./DISClib) a los scripts MVC en la carpeta [App](./App).
* Implemente las funciones que faltan de acuerdo con las necesidades del proyecto en los scripts MVC en el folder [App](./App).
* Evalúe la implementación de los scripts MVC, registre sus pruebas y análisis en los documentos de la carpeta [Docs](./Docs) (El informe **DEBE SER** en formato PDF).

## Contribuir

Las contribuciones son las que hacen que la comunidad de código abierto sea un lugar increíble para aprender, inspirar y crear. **Agradecemos cualquier** contribución que hagas.

Si tiene una sugerencia para mejorar el proyecto, bifurque (Fork) el repositorio y cree una solicitud de extracción (Pull Request). También puede simplemente abrir un problema con la etiqueta "mejora".
¡No olvides resaltar el proyecto marcando la estrella! ¡Gracias de nuevo!

1. Bifurcar (Fork) el proyecto.
2. Crear la rama (Branch) de la funcionalidad (`git checkout -b funcionalidad/increibleFuncionalidad`).
3. Confirme (Commit) sus cambios (`git commit -m adicionar una nueva funcionalidad'`).
4. Empuje (Push) los cambios a la rama (`git push origin funcionalidad/increibleFuncionalidad `).
5. Abrir una solicitud de extracción (Pull Request).
